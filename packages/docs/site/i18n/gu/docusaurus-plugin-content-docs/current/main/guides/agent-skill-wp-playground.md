# વર્ડપ્રેસ પ્લેગ્રાઉન્ડ એજન્ટ કૌશલ્યનો ઉપયોગ {#using-wordpress-playground-agent-skill}

શું તમે એવો AI સહાયક ઈચ્છો છો જેને પહેલેથી જ વર્ડપ્રેસ ઇન્સ્ટન્સ શરૂ કરવું, બ્લુપ્રિન્ટ્સ ચલાવવું અને પ્લગઇન્સ ડીબગ કરવું આવડતું હોય? wp-playground એજન્ટ કૌશલ્ય કોડિંગ એજન્ટને વર્ડપ્રેસ પ્લેગ્રાઉન્ડ CLI અને બ્રાઉઝર વર્કફ્લો શીખવે છે. તમે સામાન્ય ભાષામાં તમારી જરૂરિયાત જણાવો છો. એજન્ટ જરૂરી કમાન્ડ્સ સંભાળી લે છે.

તમારો કોડિંગ એજન્ટ જવાબ આપતા પહેલા કૌશલ્ય સંદર્ભ દસ્તાવેજ વાંચે છે — જેમાં CLI ફ્લેગ્સ, પ્રક્રિયાઓ અને ટ્રબલશૂટિંગ પગલાં સામેલ હોય છે. આથી ખાતરી થાય છે કે પ્લેગ્રાઉન્ડ કમાન્ડ્સ યોગ્ય રીતે ચલાવે છે.

## પૂર્વઆવશ્યકતાઓ {#prerequisites}

સ્કિલ ઇન્સ્ટોલ કરતા પહેલાં ખાતરી કરો કે તમારી પાસે નીચેની આવશ્યકતાઓ ઉપલબ્ધ છે:

| આવશ્યકતા  | લઘુત્તમ આવૃત્તિ       | તપાસ કમાન્ડ     |
| --------- | --------------------- | --------------- |
| Node.js   | 20.18                 | `node -v`       |
| npm / npx | Node.js સાથે સમાવેશિત | `npx --version` |

તમને એવો કોડિંગ એજન્ટ પણ જરૂરી છે જે agent skills ને સપોર્ટ કરે છે: Antigravity, Claude Code, Codex, Copilot, Cursor અથવા Gemini CLI. ખાતરી કરો કે તમારો CLI અથવા IDE નવીનતમ (latest) આવૃત્તિ પર ચાલી રહ્યો છે. આઉટપુટની ગુણવત્તા તમે પસંદ કરેલા મોડેલ પર આધારિત રહેશે.

## ઇન્સ્ટોલેશન {#installation}

### 1. ટર્મિનલ દ્વારા ઇન્સ્ટોલ કરો {#install-via-terminal}

`npx skills` CLI નો ઉપયોગ કરીને સ્કિલ ઇન્સ્ટોલ કરો:

```bash
npx skills add wordpress/agent-skills --skill wp-playground
```

### 2. મેન્યુઅલી ઇન્સ્ટોલ કરો {#install-manually}

```bash
# agent-skills ને ક્લોન કરો
git clone https://github.com/WordPress/agent-skills.git
cd agent-skills

# ડિસ્ટ્રિબ્યુશન બનાવો
node shared/scripts/skillpack-build.mjs --clean

# તમારા WordPress પ્રોજેક્ટમાં ઇન્સ્ટોલ કરો
node shared/scripts/skillpack-install.mjs --dest=../your-wp-project --targets=codex,vscode,claude,cursor,antigravity,gemini
```

આ skills ને નીચેની જગ્યાએ કોપી કરે છે:

-  VS Code / GitHub Copilot માટે `.github/skills/`
-  Claude Code માટે `.claude/skills/`
-  Cursor માટે `.cursor/skills/`
-  Antigravity માટે `.agent/skills/`
-  Gemini CLI માટે `.gemini/skills/`
-  Codex માટે `.codex/skills/`

તમારા એજન્ટ માટે સ્કિલ ડિરેક્ટરી અસ્તિત્વમાં છે કે નહીં તે તપાસીને ઇન્સ્ટોલેશનની ખાતરી કરો:

| એજન્ટ          | સ્કિલ ડિરેક્ટરી                 |
| -------------- | ------------------------------- |
| Claude Code    | `.claude/skills/wp-playground/` |
| Gemini CLI     | `.gemini/skills/wp-playground/` |
| GitHub Copilot | `.github/skills/wp-playground/` |
| Cursor         | `.cursor/skills/wp-playground/` |
| Antigravity    | `.agent/skills/wp-playground/`  |
| Codex          | `.codex/skills/wp-playground/`  |

કેટલાક એજન્ટ્સ સીધા skills ની સૂચિ બતાવવાનું પણ સપોર્ટ કરે છે:

```bash
# Claude Code
claude /skills

# Gemini CLI
gemini /skills list
```

## ટર્મિનલમાં સ્કિલનો ઉપયોગ કરો {#use-the-skill-in-the-terminal}

સ્કિલ ઇન્સ્ટોલ કર્યા પછી, તમારા કોડિંગ એજન્ટને તમારા વર્ડપ્રેસ પર્યાવરણ વિશે વર્ણન કરો. એજન્ટ બ્લુપ્રિન્ટ બનાવશે, CLI કમાન્ડ્સ ચલાવશે અને સર્વર શરૂ કરશે.

ટર્મિનલમાં તમારો કોડિંગ એજન્ટ ખોલો અને તમારી વિનંતી લખો:

```
> મારી plugin માઉન્ટ કરીને WordPress ઇન્સ્ટન્સ ચલાવો
```

એજન્ટ સ્કિલ રેફરન્સ વાંચે છે, તમારા પ્રોજેક્ટ લેઆઉટને ઓળખે છે અને `server --auto-mount` ચલાવે છે. ઇન્સ્ટન્સ `http://localhost:9400` પર શરૂ થાય છે.

### તરત જ કન્ટેન્ટ બનાવવું {#generating-content-on-the-fly}

ટેસ્ટિંગ અથવા ડેમો માટે સેમ્પલ ડેટાની જરૂર છે? તમને જે કન્ટેન્ટ સ્ટ્રક્ચર જોઈએ છે તે વર્ણન કરો:

```
> 10 પ્રકાશિત પોસ્ટ્સ સાથે WordPress ચલાવો  
```

એજન્ટ `runPHP` સ્ટેપ સાથે Blueprint બનાવે છે, જે `wp_insert_post()` નો ઉપયોગ કરીને પોસ્ટ્સ બનાવે છે.

વધુ ઉદાહરણો:

```
> 3 યુઝર્સ સાથે વર્ડપ્રેસ ચલાવો જ્યાં દરેક યુઝર પાસે 3 પોસ્ટ્સ હોય
```

```
> 5 પેજીસ અને તમામને લિંક કરતું કસ્ટમ મેનૂ સાથે વર્ડપ્રેસ ઇન્સ્ટન્સ શરૂ કરો
```

```
> 4 કેટેગરીમાં 20 પોસ્ટ્સ ધરાવતી વર્ડપ્રેસ સાઇટ બનાવો
```

દરેક પ્રોમ્પ્ટ સંપૂર્ણ બ્લુપ્રિન્ટ બનાવે છે જે લોકલી ચલાવે છે. તે બ્લુપ્રિન્ટ સ્ટેપ્સ દ્વારા યુઝર બનાવવું, રોલ સોંપવું, પોસ્ટ જનરેટ કરવી અને ટેક્સોનોમી સેટઅપ કરવાનું કામ સંભાળે છે.

### વર્ઝન સુસંગતતા પરીક્ષણ {#version-compatibility-testing}

શું તમારો પ્લગિન જૂના PHP વર્ઝન પર કામ કરે છે? સીધું પૂછો:

```
> વર્ડપ્રેસ 6.3 અને PHP 7.4 પર મારું પ્લગિન ટેસ્ટ કરો
```

```
> નવીનતમ વર્ડપ્રેસ નાઇટલી સાથે PHP 8.5 પર મારી થીમ ચલાવો
```

એજન્ટ તમારી વિનંતી મુજબ `--wp` અને `--php` ફ્લેગ્સ ઉમેરે છે. સામાન્ય પરિસ્થિતિઓ:

| પરિસ્થિતિ             | શું પૂછવું                                                           |
| ------------------  | ----------------------------------------------------------------  |
| નવીનતમ સ્થિર વર્ઝન    | "વર્ડપ્રેસ ઇન્સ્ટન્સ ચલાવો" (ડિફોલ્ટ રીતે નવીનતમ WP અને PHP 8.3)       |
| ન્યૂનતમ સપોર્ટેડ        | "વર્ડપ્રેસ 6.3 અને PHP 7.4 પર મારું પ્લગિન ટેસ્ટ કરો"                  |
| આવનારી રિલીઝ       | "વર્ડપ્રેસ નાઇટલી build ચલાવો"                                   |
| જૂનું PHP             | "PHP 7.4 સાથે વર્ડપ્રેસ શરૂ કરો"                                   |

### જટિલ પરિસ્થિતિઓ {#complex-scenarios}

એક જ પ્રોમ્પ્ટમાં અનેક આવશ્યકતાઓ જોડો:

```
> WooCommerce સાથે, 3 પ્રોડક્ટ કેટેગરી,  
  10 સેમ્પલ પ્રોડક્ટ્સ અને 2 કસ્ટમર એકાઉન્ટ ધરાવતી વર્ડપ્રેસ સાઇટ બનાવો — PHP 8.2 પર ચલતી
```

```
> મારી plugin માઉન્ટ કરીને, debug mode સક્રિય કરીને,  
  મુખ્ય છબી ધરાવતી 5 ટેસ્ટ પોસ્ટ્સ સાથે વર્ડપ્રેસ ઇન્સ્ટન્સ ચલાવો
```

એજન્ટ આ વિનંતીઓને યોગ્ય બ્લુપ્રિન્ટ સ્ટેપ્સ અને CLI ફ્લેગ્સના ક્રમમાં વહેંચે છે. દરેક વિનંતી સંપૂર્ણ રીતે કોન્ફિગર થયેલ અને ચાલી રહેલ ઇન્સ્ટન્સ તૈયાર કરે છે.

## સ્કિલ કેવી રીતે કાર્ય કરે છે {#how-the-skill-works}

wp-playground સ્કિલ Markdown ફાઇલોનો સમૂહ છે, જેને તમારો કોડિંગ એજન્ટ તમારા વિનંતી પ્લેગ્રાઉન્ડ સંબંધિત હોય ત્યારે તેના કન્ટેક્સ્ટમાં લોડ કરે છે. આ સ્કિલમાં નીચેની ફાઇલો સામેલ છે:

* **SKILL.md** — મુખ્ય પ્રક્રિયા: માર્ગદર્શિકા (guardrails), પગલું-દર-પગલું વર્કફ્લો, વેરિફિકેશન ચેક અને નિષ્ફળતા પરિસ્થિતિઓ.
* **references/cli-commands.md** — દરેક ફ્લેગ અને ડિફોલ્ટ મૂલ્ય સાથે CLI ચીટશીટ.
* **references/blueprint.md** — બ્લુપ્રિન્ટ ની રચના, સામાન્ય સ્ટેપ્સ અને લખાણ (authoring) સૂચનો.
* **references/debugging.md** — Xdebug સેટઅપ, પાથ મેપિંગ અને ટ્રબલશૂટિંગ પગલાં.

તમારો કોડિંગ એજન્ટ કમાન્ડ્સ જનરેટ કરતા પહેલા આ ફાઇલો વાંચે છે, જેથી યોગ્ય ફ્લેગ્સ ઉપયોગમાં લેવાય અને સામાન્ય ભૂલોથી તમને ચેતવણી મળે.

## આગળના પગલાં {#next-steps}

* **પ્લગઈન ડેવલપર્સ માટે વર્ડપ્રેસ પ્લેગ્રાઉન્ડ માર્ગદર્શિકા** — પ્લેગ્રાઉન્ડ સાથે પ્લગિનનું પ્રદર્શન અને વિકાસ કરો
* **થીમ ડેવલપર્સ માટે વર્ડપ્રેસ પ્લેગ્રાઉન્ડ માર્ગદર્શિકા** — પ્લેગ્રાઉન્ડ નો ઉપયોગ કરીને થીમ બનાવો અને ડેમો આપો
* **અપસ્ટ્રીમ પ્લેગ્રાઉન્ડ દસ્તાવેજીકરણ** — API, આર્કિટેક્ચર અને એડવાન્સ કોન્ફિગરેશન માટે સંપૂર્ણ સંદર્ભ ([https://wordpress.github.io/wordpress-playground/](https://wordpress.github.io/wordpress-playground/))

જો તમે ઇચ્છો તો હું આખું દસ્તાવેજ એકસાથે સચોટ રીતે ફોર્મેટ કરીને પણ આપી શકું.
