## ウェブサイトに WordPress を埋め込む

<!--
## Embed WordPress on your website
-->

Playground は、次のように HTML `<iframe>` タグを使用して Web サイトに埋め込むことができます。

<!--
Playground can be embedded on your website using the HTML `<iframe>` tag as follows:
-->

```html
<iframe src="https://playground.wordpress.net/"></iframe>
```

訪問者全員にプライベート WordPress インスタンスが無料で提供されます。その後、[Playground API](/developers/apis/)のいずれかを使用してカスタマイズできます。

## あらゆるプラグインやテーマを紹介

<!--
## Showcase any plugin or theme
-->

プラグインが WordPress ディレクトリにない場合はどうなるでしょうか?

<!--
What if your plugin is not in the WordPress directory?
-->

[JSON ブループリント](/blueprints) を使えば、Playground で公開することも可能です。例えば、以下のブループリントは、ウェブサイトからプラグインとテーマをダウンロードしてインストールし、スターターコンテンツもインポートします。

<!--
You can still showcase it on Playground by using [JSON Blueprints](/blueprints). For example, this Blueprint would download and install a plugin and a theme from your website and also import some starter content:
-->

```json
{
	"steps": [
		{
			"step": "installPlugin",
			"pluginData": {
				"resource": "url",
				"url": "https://your-site.com/your-plugin.zip"
			}
		},
		{
			"step": "installTheme",
			"themeData": {
				"resource": "url",
				"url": "https://your-site.com/your-theme.zip"
			}
		},
		{
			"step": "importWxr",
			"pluginData": {
				"resource": "url",
				"url": "https://your-site.com/starter-content.wxr"
			}
		}
	]
}
```

詳細については、[ブループリントの使用を開始する](/blueprints/getting-started)を参照してください。

<!--
See [getting started with Blueprints](/blueprints/getting-started) to learn more.
-->

## リポジトリからのプルリクエストをプレビューする

<!--
## Preview pull requests from your repository
-->

[Gutenberg PR プレビューアーのライブ例](https://playground.wordpress.net/gutenberg.html)をご覧ください。

<!--
See the [live example of Gutenberg PR previewer](https://playground.wordpress.net/gutenberg.html).
-->

次の場合には、プレイグラウンド を Pull Request プレビューアーとして使用できます。

<!--
You can use Playground as a Pull Request previewer if:
-->

-   WordPress プラグインまたはテーマが CI パイプラインを使用している
-   CI パイプラインがプラグインまたはテーマをバンドルしている
-   CI パイプラインによって生成された zip ファイルを公開できる

<!--
-   Your WordPress plugin or theme uses a CI pipeline
-   Your CI pipeline bundles your plugin or theme
-   You can expose the zip file generated by your CI pipeline publicly
-->

これらの zip バンドルは通常の WordPress プラグインと何ら変わりません。つまり、[JSON Blueprints](/blueprints) API を使ってプレイグラウンドにインストールできます。https://your-site.com/pull-request-1234.zip のようなエンドポイントを公開すれば、あとは以下のブループリントが処理してくれます。

<!--
Those zip bundles aren't any different from regular WordPress Plugins, which means you can install them in Playground using the [JSON Blueprints](/blueprints) API. Once you exposed an endpoint like https://your-site.com/pull-request-1234.zip, the following Blueprint will do the rest:
-->

```json
{
	"steps": [
		{
			"step": "installPlugin",
			"pluginData": {
				"resource": "url",
				"url": "https://your-site.com/pull-request-1234.zip"
			}
		}
	]
}
```

公式の プレイグラウンド デモでは、この手法を使用して、Gutenberg リポジトリからのプル リクエストをプレビューします。

## 互換性テスト環境を構築する

<!--
## Build a compatibility testing environment
-->

設定可能な PHP と WordPress を備えたライブ プラグイン デモは、優れた互換性テスト環境になります。

<!--
A live plugin demo with a configurable PHP and WordPress makes an excellent compatibility testing environment.
-->

クエリ API を使用すると、URL に `php` および `wp` クエリ パラメータを追加するだけです。

<!--
With the Query API, you'd simply add the `php` and `wp` query parameters to the URL:
-->

```html
<iframe src="https://playground.wordpress.net/?php=8.3&wp=6.1"></iframe>
```

JSON ブループリントでは、`preferredVersions` プロパティを使用します。

<!--
With JSON Blueprints, you'd use the `preferredVersions` property:
-->

```json
{
	"preferredVersions": {
		"php": "8.3",
		"wp": "6.1"
	}
}
```

## ブラウザで PHP コードを実行する

<!--
## Run PHP code in the browser
-->

JavaScript API には、ブラウザで PHP コードを実行するために使用できる `run()` メソッドが用意されています。

<!--
The JavaScript API provides the `run()` method which you can use to run PHP code in the browser:
-->

```html
<iframe id="wp"></iframe>
<script type="module">
	const client = await startPlaygroundWeb({
		iframe: document.getElementById('wp'),
		remoteUrl: 'https://playground.wordpress.net/remote.html',
	});
	await client.isReady;
	await client.run({
		code: `<?php
		require("/wordpress/wp-load.php");

		update_option("blogname", "Playground is really cool!");
		echo "Site title updated!";
		`,
	});
	client.goTo('/');
</script>
```

これを Monaco や CodeMirror などのコード エディターと組み合わせると、[この記事](https://adamadam.blog/2023/02/16/how-to-modify-html-in-a-php-wordpress-plugin-using-the-new-tag-processor-api/) のようなライブ コード スニペットを取得できます!

-->
